# Plan: Bulk User Import

## Execution Strategy

- Approach: sequential phases, parallel tasks within Phase 2 where dependencies allow
- Estimated workers: 2
- Estimated validators: 1

## Work Phases

### Phase 1: Database Foundation
- **Goal:** Add the `import_batch_id` column to the users table.
- **Tasks:** T-001
- **Dependencies:** None
- **Validation checkpoint:** No (simple migration, validated at Phase 2 checkpoint)

### Phase 2: Core Import Logic
- **Goal:** Implement CSV parsing, validation pipeline, and import executor.
- **Tasks:** T-002, T-003, T-004, T-005
- **Dependencies:** Phase 1 complete
- **Validation checkpoint:** Yes — validate core logic before wiring up the endpoint

### Phase 3: API Integration
- **Goal:** Wire up the upload endpoint and error report generation.
- **Tasks:** T-006
- **Dependencies:** Phase 2 complete
- **Validation checkpoint:** Yes — final validation after all tasks complete

## Dependency Map

| Task/Phase | Depends On | Blocking? |
|---|---|---|
| T-001 (migration) | Nothing | Yes — blocks all other tasks |
| T-002 (parser) | T-001 | Yes — blocks T-003, T-004 |
| T-003 (structural validator) | T-002 | Yes — blocks T-005 |
| T-004 (business validator) | T-002 | Yes — blocks T-005 |
| T-005 (import executor) | T-003, T-004 | Yes — blocks T-006 |
| T-006 (endpoint + error report) | T-005 | No — final task |

## Integration Approach

Incremental integration. Each phase builds on the previous:
1. Migration provides the schema.
2. Core logic components are tested independently, then wired together.
3. Endpoint integrates all components and exposes the feature via API.

## Rollout Order

1. Run database migration (deploy migration first).
2. Deploy application code with new endpoint.
3. Configure error report cleanup cron.

## Risks / Coordination Notes

- Worker 2 has significantly more work than Worker 1 during Phase 2. If Worker 1 finishes T-001 quickly, consider re-assigning a Phase 2 task to balance load (requires approval).
- Phase 2 validation checkpoint is important — catching issues in core logic before endpoint integration saves rework.
